## 1. 安装与启动

### 方式一：EXE 分发版（推荐普通用户）

1. 前往 [GitHub Releases](https://github.com/PegSkyWalf/Stellaris-Custom-GUI-Editor/releases) 下载最新版 ZIP 文件
2. **将 ZIP 完整解压**到你希望放置软件的目录（如 `D:\Tools\StellarisGUIEditor\`）
3. 双击 `StellarisGUIEditor.exe` 启动

> ⚠️ **必须解压后再运行，不能直接在压缩包内双击执行。**  
> EXE 旁边的 `_internal\` 目录包含所有运行时库，缺少它将无法启动。

### 方式二：从源码运行（推荐开发者）

```bash
# 需要 Python 3.10+（https://www.python.org/downloads/）
git clone https://github.com/PegSkyWalf/Stellaris-Custom-GUI-Editor.git
cd Stellaris-Custom-GUI-Editor
pip install -r requirements.txt
python main.py
```

## 2. 首次配置向导

首次启动时，向导自动弹出，引导你完成三步配置：

### 步骤 1：配置游戏目录

点击 **自动检测**：编辑器通过 Steam 注册表自动定位群星安装目录。

如果自动检测失败，点击 **浏览…** 手动选择：
- Steam 用户：右键群星 → 管理 → 浏览本地文件
- 典型路径示例：`E:\SteamLibrary\steamapps\common\Stellaris`
- **验证标准**：路径下应存在 `interface\` 子目录

> 也可以选择性地配置一个或多个 **模组目录**（Mod 文件夹根目录），配置后编辑器可以加载模组的精灵图、本地化文本和按钮效果。

### 步骤 2：选择界面主题

| 主题 | 适合场景 |
|------|---------|
| 深色（默认） | 长时间使用，护眼 |
| 浅色 | 强光环境，高对比度 |
| 深海蓝 | 专业蓝调风格 |

可在向导完成后随时在 **工具 → 设置 → 主题** 中更改，也可自定义强调色。

### 步骤 3：完成

点击 **开始使用**，进入主界面。编辑器开始在后台异步加载游戏资源（底部状态栏显示进度，过程中界面可正常操作）。

## 4. 基本工作流

### 4.1 新建 GUI 文件

1. 菜单 → **文件 → 新建 GUI 文件**
2. 输入文件名（如 `my_custom_window.gui`）和顶层容器名称
3. 编辑器创建一个空的 `containerWindowType` 作为起始结构

### 4.2 打开已有文件

- 菜单 → **文件 → 打开**，选择 `.gui` 文件
- 或在左侧**文件浏览**面板中双击文件名

### 4.3 添加控件

1. 在左侧**控件库**面板中找到需要的控件类型
2. 双击，或拖拽到画布的目标容器中
3. 控件出现在画布上，同时在图层面板中出现对应节点

### 4.4 选择与编辑控件

- **点击画布**上的控件直接选中
- 或点击**图层面板**中的对应节点
- 选中后，右侧**属性面板**显示该控件的所有属性，可直接编辑

### 4.5 移动控件

- **拖拽**：在画布上拖拽选中的控件
- **键盘微移**：选中后按方向键（每次 1 像素），Shift+方向键（每次 10 像素）
- **属性面板**：直接修改 `position` 的 `x`、`y` 值

### 4.6 调整控件大小

- **拖拽手柄**：选中控件后，在边框上出现 8 个调整手柄，拖拽调整大小
- **属性面板**：直接修改 `size` 的 `width`、`height` 值

### 4.7 保存

- **Ctrl+S** 保存到原文件
- 菜单 → **文件 → 另存为** 保存到新路径

> 编辑器输出的 `.gui` 文件格式与游戏原版完全兼容，可直接放入 Mod 的 `interface/` 目录使用。

## 5. 画布操作详解

### 5.1 视口控制

| 操作 | 方法 |
|------|------|
| **缩放** | Ctrl + 鼠标滚轮（以鼠标所在位置为中心） |
| **平移** | 鼠标中键拖拽 / Alt + 左键拖拽 |
| **适应画布** | Ctrl+0（将整个画布缩放到可见范围） |
| **缩放到 100%** | 菜单 → 视图 → 实际大小 |

缩放范围：**25% – 400%**

### 5.2 选择控件

| 操作 | 方法 |
|------|------|
| 单选 | 点击控件 |
| 多选 | Ctrl + 点击，或框选（空白处拖拽） |
| 取消选择 | 点击空白处 |
| 选中所有 | Ctrl+A |

### 5.3 预览模式

按 **Ctrl+P** 进入/退出预览模式：
- 预览模式下：隐藏所有框线、类型标签、选择手柄
- 画布只显示精灵图和布局，近似游戏内效果
- 预览模式下不可编辑控件

### 5.4 撤销与重做

| 操作 | 快捷键 |
|------|--------|
| 撤销 | Ctrl+Z |
| 重做 | Ctrl+Y |

支持的可撤销操作：移动控件、修改属性、添加控件、删除控件、复制控件。

## 6. 群星坐标系简介

理解 Stellaris 的坐标系有助于正确设置 `position`、`orientation`、`origo` 等属性。

### 6.1 基本概念

| 属性 | 含义 | 默认值 |
|------|------|--------|
| `orientation` | 控件锚点相对于**父容器**的位置 | `UPPER_LEFT` |
| `origo` | 控件自身的参考点 | `UPPER_LEFT` |
| `position` | 从父容器锚点到控件参考点的偏移量（像素） | `{ x=0 y=0 }` |

### 6.2 方向值

| 值 | 含义 |
|----|------|
| `UPPER_LEFT` | 左上角（默认） |
| `UPPER_RIGHT` | 右上角 |
| `LOWER_LEFT` | 左下角 |
| `LOWER_RIGHT` | 右下角 |
| `CENTER` | 中心 |
| `CENTER_UP` | 上边中央 |
| `CENTER_DOWN` | 下边中央 |
| `LEFT_UP` / `LEFT_DOWN` | 左边上/下 |
| `RIGHT_UP` / `RIGHT_DOWN` | 右边上/下 |

### 6.3 定位示例

**居中放置一个控件**（在其父容器中心）：
```
orientation = center
origo = center
position = { x = 0 y = 0 }
```

**在父容器右下角放置**（偏移 -10, -10）：
```
orientation = LOWER_RIGHT
origo = LOWER_RIGHT
position = { x = -10 y = -10 }
```

### 6.4 精灵图尺寸规则

| 精灵类型 | 尺寸控制 | 视觉缩放 |
|---------|---------|---------|
| `spriteType` | 精灵图本身固定尺寸，`size` 属性影响点击区域 | 用 `scale` 调整视觉大小 |
| `quadTextureSprite` / `corneredTileSpriteType` | `size` 属性直接控制显示尺寸 | 9 块拉伸，角不变形 |

## 7. 控件类型说明

### containerWindowType — 容器窗口

群星 GUI 的基本组织单元。所有其他控件必须放在容器内。

- 可无限嵌套
- `background` 是其可选子属性（非独立控件），用于设置容器背景精灵图
- 声明 `size` 时有明确边界；不声明 `size` 时（隐式容器），编辑器自动以子控件包围盒确定显示范围

**必要属性：** `name`

---

### iconType — 图标

显示静态精灵图。

- 使用 `spriteType` 引用固定尺寸精灵图
- 使用 `quadTextureSprite` 引用可拉伸精灵图（需指定 `size`）
- 不可交互

---

### buttonType — 按钮

使用游戏内置硬编码动作的按钮。

- 动作通过 `buttonText` 或 `buttonFont` 等属性定义
- 点击效果固定（如关闭窗口、确认等）

---

### effectButtonType — 效果按钮

Custom GUI 的核心控件，允许绑定自定义 `button_effect` 脚本。

**关键属性：**

| 属性 | 说明 |
|------|------|
| `effect` | 关联的 button_effect 名称（在 `common/button_effects/` 中定义） |
| `buttonText` | 按钮显示文本（支持本地化键） |
| `buttonFont` | 字体名称 |
| `format` | 文本对齐：`CENTER`、`CENTER_LEFT`、`left`、`right` 等 |
| `text_offset` | 文本相对于按钮的偏移量 `{ x=0 y=0 }` |
| `tooltipText` | 鼠标悬停时显示的提示文本（本地化键） |
| `spriteType` | 按钮外观精灵图 |
| `scale` | 整体缩放（不影响 `size` 属性） |
| `alwaysTransparent` | 设为 `yes` 时按钮透明（仅保留点击区域） |

---

### instantTextBoxType — 即时文本框

显示文本内容，支持本地化键。

**关键属性：** `text`（或本地化键）、`font`、`maxWidth`、`format`

---

### textBoxType — 文本框

支持多行文本显示。

---

### editBoxType — 输入框

允许玩家输入文本。

---

### checkBoxType — 复选框

二值状态切换控件。

---

### scrollAreaType — 滚动区域

可以包含超出其 `size` 的子控件，超出部分通过滚动条访问。

## 8. 属性面板详解

### 8.1 通用属性

| 属性名 | 说明 |
|--------|------|
| `name` | 控件唯一名称（同一容器内不应重复） |
| `position` | 位置偏移（见第 6 章） |
| `size` | 显示/交互尺寸（宽 × 高） |
| `orientation` | 父容器锚点位置 |
| `origo` | 控件自身参考点 |
| `clipping` | `yes`：裁剪超出范围的子控件 |
| `moveable` | `yes`：玩家可在游戏内拖动此窗口 |

### 8.2 精灵图属性

| 属性名 | 说明 |
|--------|------|
| `spriteType` | 引用 `.gfx` 中定义的 spriteType 名称 |
| `quadTextureSprite` | 引用 `.gfx` 中定义的 corneredTileSpriteType 名称 |
| `scale` | 视觉缩放系数（`1.0` = 原大小） |

点击属性行右侧的 **"选择精灵…"** 按钮，可直接在精灵图库中选择并应用。

### 8.3 文本属性

| 属性名 | 说明 |
|--------|------|
| `buttonText` | 按钮文本内容（直接文本或本地化键） |
| `buttonFont` | 字体（如 `CeleTbR_cg_16b`、`standard` 等） |
| `format` | 对齐方式：`CENTER`、`CENTER_LEFT`、`left`、`right` |
| `text_offset` | 文本偏移量 `{ x=0 y=0 }` |

当文本内容是本地化键时，属性面板会在输入框下方显示对应的翻译文本（如果已加载本地化文件）。

### 8.4 原始属性编辑器

属性面板下方的**原始属性**区域以键值表格形式显示所有属性（包括编辑器没有专用控件的属性）。

在这里可以：
- 查看全部属性
- 添加任意自定义属性键值
- 修改不在常用字段中的属性

> 原始属性编辑器确保保存时不丢失任何属性，即使是编辑器不认识的属性也会原样保留。

## 9. 图层面板

图层面板以树形结构展示当前 GUI 文件的完整控件层次。

### 基本操作

| 操作 | 方法 |
|------|------|
| 选中控件 | 点击节点（画布同步高亮） |
| 展开/折叠容器 | 点击左侧三角箭头 |
| 切换可见性 | 点击节点右侧眼睛图标 |
| 多选 | Ctrl + 点击 |

### 可见性说明

图层面板的**可见性**（眼睛图标）是**编辑器内的本地显示控制**，不影响 `.gui` 文件的实际内容，也与虚拟编组的可见性完全独立——两套可见性互不干扰。

## 10. 虚拟编组

虚拟编组是一套**独立于 `.gui` 文件结构**的控件分组工具，方便你按功能区域管理复杂界面。

### 特点

- 编组信息保存在侧车文件 `*.groups.json` 中，不修改 `.gui` 文件
- 可将任意控件加入一个或多个编组
- 切换编组可见性不影响图层面板的单独控件可见性（两套独立运作）

### 使用方式

1. 在下方切换到**虚拟编组**面板
2. 点击 **新建编组**，输入名称（如"背景图层"、"按钮区域"）
3. 在图层面板或画布中右键控件 → **加入编组**
4. 点击编组名称旁的眼睛图标，一键隐藏/显示整个编组

## 11. 对齐与分布工具

通过 **编辑 → 对齐** 菜单访问，所有操作均可撤销（Ctrl+Z）。

### 对齐（需选中 2 个以上控件）

| 菜单项 | 行为 |
|--------|------|
| 左对齐 | 所有控件左边缘对齐到最左侧控件 |
| 右对齐 | 所有控件右边缘对齐到最右侧控件 |
| 顶对齐 | 所有控件顶边对齐到最高控件 |
| 底对齐 | 所有控件底边对齐到最低控件 |
| 水平居中 | 所有控件水平中轴对齐到平均中心 |
| 垂直居中 | 所有控件垂直中轴对齐到平均中心 |

### 均匀分布（需选中 3 个以上控件）

| 菜单项 | 行为 |
|--------|------|
| 水平间距均匀 | 在最左和最右控件之间均匀排布其余控件 |
| 垂直间距均匀 | 在最顶和最底控件之间均匀排布其余控件 |

### 相同尺寸（需选中 2 个以上控件）

| 菜单项 | 行为 |
|--------|------|
| 相同宽度 | 所有控件宽度设为最大宽度 |
| 相同高度 | 所有控件高度设为最大高度 |
| 相同尺寸 | 宽高同时统一 |

## 12. 阵列与镜像

通过 **编辑 → 阵列与镜像** 访问。所有操作均生成新副本（不破坏原件）并可撤销。  
**副本命名规则**：所有生成的副本自动追加 `_1`、`_2` 等序号后缀，不会产生重复名称。

### 12.1 线性阵列

将选中控件沿指定方向复制 N 份，每份相对前一份偏移固定距离。

| 参数 | 说明 |
|------|------|
| 复制数量 | 生成的副本数（不含原件） |
| X 偏移 | 每份水平偏移（像素，负值向左） |
| Y 偏移 | 每份垂直偏移（像素，负值向上） |

**示例**：X 偏移 80、复制 3 份 → 在原件右侧每隔 80px 生成 3 个副本。

### 12.2 圆形阵列

在圆形轨迹上均匀分布副本，提供两种模式：

#### 圆心模式（推荐入门使用）

- 选中控件保持不动，作为圆心
- 在指定半径的圆环上均匀放置 N 个副本
- **无需手动输入圆心坐标**，编辑器自动以选中控件的几何中心为圆心

#### 环上模式

- 选中控件作为环上 angle=0 的起点位置（不移动）
- 其余 N 个副本均匀分布在圆环剩余位置，共 N+1 个点闭合成环
- 需要手动输入圆心坐标（可输入参考容器的中心坐标）

**实时预览**：对话框右侧有 200×200 预览图，橙色方块为原件位置，灰色方块为副本位置，虚线圆为分布轨迹。

### 12.3 镜像

以选中控件集合的**包围盒中心**为轴进行镜像。

| 参数 | 说明 |
|------|------|
| 镜像轴 | 垂直轴（左右翻转）/ 水平轴（上下翻转） |
| 保留原件 | 勾选：镜像复制（原件+镜像副本）；不勾选：原件移动到镜像位置 |

> ⚠️ 群星的 `.gui` 文件不支持旋转/翻转属性，镜像操作仅改变控件的 **position**，视觉上的对称效果需配合对称的精灵图资源。

## 13. 吸附对齐辅助线

### 吸附类型

| 类型 | 触发条件 |
|------|---------|
| 边缘对齐 | 被拖控件的任意边缘与其他控件的边缘距离在阈值内 |
| 中轴对齐 | 被拖控件的水平/垂直中轴与其他控件的中轴对齐 |

### 设置

前往 **工具 → 设置 → 画布** 可调整：
- **吸附阈值**（像素）：默认 8px，越小越精确
- **显示辅助线**：关闭后不显示线条但仍吸附
- 按住 **Alt** 拖拽可临时禁用吸附

## 14. 重复名称警告面板

底部停靠，与代码视图同一排标签页。

### 功能

- 实时扫描当前文档所有控件，找出 **`name` 属性重复** 的控件
- 以树形列表展示：父项为重复的名称 + 出现次数，子项为每个出现位置的结构路径
- **点击子项**：画布自动选中该控件并滚动到可见区域

### 何时出现警告

- 手动将两个控件改为同名
- 复制控件后未修改名称（正常操作下编辑器会自动添加序号，手动编辑代码视图除外）
- 从其他编辑器导入的 `.gui` 文件中本就存在重复名称

### 注意

重复名称不会阻止保存，仅作提示。但群星本身对重名控件的行为未定义，可能引起界面显示或脚本访问的不确定性。

## 16. 代码视图

代码视图实时显示当前 GUI 文档对应的 PDX Script 代码。

### 精准高亮定位

在画布或图层面板中选中某控件时，代码视图会**自动滚动并高亮该控件对应的代码块**。

定位算法基于**控件结构路径**（类型 + 名称 + 在父容器中的索引），而非简单的名称搜索。这意味着：即使两个控件重名（如不同容器下各有一个叫 `portrait` 的 iconType），也能精准定位到正确的那一个。

### 直接编辑代码

代码视图不只是只读的，你可以**直接在代码视图中修改代码**：
1. 修改代码后，编辑器自动解析
2. 成功解析后，画布实时更新
3. 解析失败时显示错误提示，原代码不受影响

## 17. 事件关联面板

事件关联面板展示游戏/模组事件文件中对当前 GUI 文件的引用关系。

当你打开一个 `.gui` 文件时，编辑器自动扫描所有事件文件，找出通过 `custom_gui = "窗口名称"` 触发该 GUI 的事件，并在此面板中列出。

**用途**：快速了解你编辑的这个 GUI 窗口在哪些事件中被使用，方便追踪和调试。

## 18. Button Effects 编辑器

Button Effects 编辑器用于可视化编辑游戏的 `common/button_effects/*.txt` 文件。

每个 `button_effect` 块包含：
- `potential`：效果的触发条件（Stellaris 脚本条件）
- `allow`：允许条件
- `effect`：效果内容（Stellaris 脚本效果）

在编辑器中，你可以浏览所有 button_effect，直接在文本框中编辑其内容，并保存回对应文件。

> **提示**：`effectButtonType` 的 `effect` 属性值就是这里定义的 button_effect 名称。

## 19. GFX 精灵注册生成器

使用自定义精灵图时，需要在 `.gfx` 文件中注册。GFX 生成器帮助你快速生成标准格式的注册代码。

使用方式：
1. 菜单 → **工具 → GFX 生成器**
2. 填写精灵图名称（如 `GFX_my_button`）
3. 选择精灵类型：`spriteType` 或 `corneredTileSpriteType`
4. 填写贴图文件路径（相对于 Mod 根目录）
5. 点击 **生成**，代码自动复制到剪贴板或显示在文本框中

生成的代码格式示例（spriteType）：
```
spriteType = {
    name = "GFX_my_button"
    texturefile = "gfx/interface/my_button.dds"
    noOfFrames = 2
}
```

## 22. 常见问题与故障排除

### 启动问题

**Q：EXE 启动报错 `Failed to load Python DLL`**  
A：你可能只复制了 `.exe` 文件，没有将 `_internal\` 文件夹一同复制。必须解压整个 ZIP 并保持文件夹结构完整。

**Q：程序提示"未配置游戏目录"**  
A：前往 **工具 → 设置 → 路径**，点击"自动检测"，或手动浏览到群星安装目录（目录下需有 `interface\` 子目录）。

### 精灵图问题

**Q：精灵图显示为棋盘格**  
A：可能原因：
1. 游戏目录配置不正确
2. 精灵图使用了 BC7 格式，需要 `pydds` 库（已包含在 EXE 分发版中；源码运行时请 `pip install pydds`）

**Q：精灵图显示但比例失真（被拉伸/压扁）**  
A：检查 `.gfx` 文件中的精灵定义，确认 `noOfFrames` 值是否正确（如果精灵图横向排列多帧动画，该值不为 1 时需要正确设置）。

### 布局/位置问题

**Q：控件在画布中的位置和游戏内不一致**  
A：可能涉及 `orientation` 和 `origo` 的设置。请参考[第 6 章](#6-群星坐标系简介)。对于有父容器的控件，位置是相对于父容器计算的，不是绝对屏幕坐标。

**Q：控件在游戏内可以看到，但在编辑器中超出了父容器范围**  
A：这可能是隐式容器（无 `size` 属性）的显示计算问题。编辑器按子控件包围盒估算容器大小，结果可能与游戏实际显示略有差异，但不影响导出的代码正确性。

### 保存/文件问题

**Q：保存后游戏内无法识别该 GUI 文件**  
A：检查：
1. 文件是否放在 Mod 的 `interface\` 目录下
2. 是否在游戏事件脚本中用 `custom_gui = "窗口名称"` 触发
3. 文件编码是否为 UTF-8（无 BOM）

**Q：程序崩溃**  
A：日志文件位于 `C:\Users\用户名\.stellaris_gui_editor\logs\`。提交 Issue 时请附上最新的 `.log` 文件内容，这有助于快速定位问题。
