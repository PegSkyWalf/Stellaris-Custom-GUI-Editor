## 1. 安装与启动

### 方式一：EXE 分发版（推荐普通用户）

1. 下载最新版 ZIP 文件（从 GitHub Releases 页面）
2. 解压到任意目录（如 `D:\Tools\StellarisGUIEditor\`）
3. 双击 `StellarisGUIEditor.exe` 启动

### 方式二：从源码运行（推荐开发者）

```bash
# 安装 Python 3.10+（https://www.python.org/）
# 克隆项目
git clone https://github.com/PegSkyWalf/Stellaris-Custom-GUI-Editor.git
cd StellarisGUIEditor

# 安装依赖
pip install -r requirements.txt

# 启动
python main.py
```

## 2. 首次配置向导

首次启动时，向导会自动弹出，引导你完成 3 步配置：

### 步骤 1：指定游戏安装目录

- 点击**自动检测**：编辑器会扫描 Steam 注册表和常见安装路径
- 或点击**浏览…**：手动找到群星安装目录
  - Steam 用户：右键群星 → 管理 → 浏览本地文件
  - 路径示例：`E:\SteamLibrary\steamapps\common\Stellaris`
  - **验证**：路径中应有 `interface/` 子目录

### 步骤 2：选择界面主题

| 主题 | 特点 |
|------|------|
| 深色（默认） | 低亮度，适合长时间使用 |
| 浅色 | 高对比度，日间光线强时清晰 |
| 深海蓝 | 专业感，蓝色调配色 |

### 步骤 3：完成

记住快速上手流程，点击**开始使用**。

> 任何时候都可以在 **工具 → 设置** 中修改这些选项。

## 4. 基本工作流

### 典型 Mod 制作流程

```
1. 打开模组目录 → 2. 新建/打开 GUI 文件 → 3. 添加控件
→ 4. 设置属性与精灵图 → 5. 保存 → 6. 在游戏中测试
→ 7. 根据测试结果调整（重复 3-6）
```

### 详细步骤

**步骤 1：打开模组目录**
- 菜单：文件 → 打开模组目录
- 选择你的 Mod 根目录（包含 `descriptor.mod` 的那个文件夹）
- 编辑器会自动加载模组的精灵图和 GFX 文件

**步骤 2：新建 GUI 文件**
- 菜单：文件 → 新建 GUI 文件
- 输入文件名（如 `my_custom_gui`）
- 输入 GUI 键名（如 `my_mod_main_window`）——这是 `custom_gui = "my_mod_main_window"` 中引用的值
- 保存目录会自动填写为模组的 `interface/` 目录

**步骤 3：添加控件**
- 在**控件库**中双击控件类型，输入名称
- 控件出现在画布中心，可立即拖动

**步骤 4：设置属性**
- 点击选中画布上的控件
- 在**属性面板**中修改位置、大小、精灵图等

**步骤 5：保存**
- Ctrl+S 保存到 Mod 目录
- 文件保存后可直接放入 Mod 的 `interface/` 目录

## 5. 画布操作详解

### 视图控制

| 操作 | 方式 |
|------|------|
| 平移视图 | 鼠标中键拖拽，或按住 Alt 再左键拖拽 |
| 缩放 | Ctrl + 鼠标滚轮 |
| 适应全部内容 | Ctrl+0 |
| 居中到选中控件 | Ctrl+F |
| 缩放到选中控件 | Ctrl+Shift+F |

### 控件操作

| 操作 | 方式 |
|------|------|
| 选中 | 左键单击 |
| 多选 | Ctrl+单击，或框选（在空白处拖拽） |
| 移动 | 拖拽选中控件 |
| 微移 | 方向键（每次 1 像素）或 Shift+方向键（每次 8 像素）|
| 调整大小 | 拖拽控件边框的蓝色手柄 |
| 复制 | Ctrl+D |
| 删除 | Delete |

### 对齐

多选控件后，使用 对齐 菜单或快捷键对齐：
- Ctrl+Alt+1/2/3：左/右/水平居中对齐
- Ctrl+Alt+4/5/6：上/下/垂直居中对齐

### 网格与吸附

- **Ctrl+G**：显示/隐藏网格
- 网格尺寸可在 工具 → 设置 → 画布 中调整（默认 8 px）
- 启用吸附后，移动控件时自动对齐到网格

### 预览模式

- **Ctrl+P**：切换预览模式
- 预览模式隐藏编辑器辅助线和手柄，近似游戏内效果
- 不可在预览模式下编辑控件

## 6. 控件类型说明

| 控件类型 | 颜色 | 用途 |
|----------|------|------|
| containerWindowType | 蓝色 | 容器，存放其他控件；顶层 GUI 窗口 |
| iconType | 绿色 | 静态图标（固定尺寸精灵图） |
| buttonType | 橙色 | 普通按钮（硬编码游戏行为） |
| effectButtonType | 红色 | 效果按钮（链接 button_effect 脚本） |
| instantTextBoxType | 紫色 | 单行/多行文本框（不可滚动） |
| textBoxType | 灰色 | 可滚动文本框 |
| editBoxType | 青绿色 | 文本输入框 |
| checkBoxType | 橙色 | 复选框 |
| listboxType | 青色 | 可滚动列表 |
| scrollAreaType | 蓝色 | 滚动区域容器 |
| gridBoxType | 红色 | 网格布局容器 |
| overlappingElementsBoxType | 紫色 | 重叠元素容器（用于多列表等） |

> **提示**：在控件库中将鼠标悬停在控件名称上可看到详细说明。

## 7. 属性面板详解

属性面板按类别分组：

### 基本信息
- **名称**：控件的唯一标识符（`name = "..."` 属性），在同级控件中必须唯一

### 位置与大小
- **position**：相对于锚点的偏移量（Stellaris 坐标系）
- **size**：控件尺寸（对 spriteType 无效，尺寸由图片决定）
- **orientation**：锚点在父元素中的位置（如 `CENTER`、`UPPER_RIGHT`）
- **origo**：本控件的哪个点对齐到锚点（默认左上角）

### 外观（精灵图）
- **spriteType**：固定尺寸精灵图，大小由图片决定，用 `scale` 缩放
- **quadTextureSprite**：可拉伸精灵图（corneredTileSpriteType），大小由 `size` 决定
- **scale**：仅对 spriteType 有效的缩放系数（默认 1.0）

### 文本内容
- **buttonText / text**：按钮文字或文本内容，填写本地化键（如 `MY_MOD_TITLE`）
- **font**：字体名称（如 `cg_16b`、`malgun_goth_24`）
- **maxWidth / maxHeight**：文本框最大尺寸

### 交互（effectButtonType 专用）
- **effect**：button_effect 脚本名称
- **tooltipText**：悬停提示的本地化键

### 背景（background 子块）
- 仅对 containerWindowType 有效
- 设置一个铺满容器的背景精灵图

### 原始属性编辑器（折叠，默认隐藏）
- 显示所有不被上方表单覆盖的额外属性
- 可直接编辑、删除任意属性，也可添加新的自定义属性
- **重要**：防止不常见属性在保存时意外丢失

## 9. 代码视图

底部**代码视图**面板：

- 实时显示当前文档的 `.gui` 代码（PDX 语法高亮）
- **编辑代码后点击"应用"**：代码回写到画布（画布刷新）
- 选中控件时对应代码自动高亮
- 适合检查生成代码是否符合预期

---

## 10. 事件关联面板

右侧**事件关联**标签页：

- 列出所有引用当前 GUI 的事件（`custom_gui = "..."` 匹配）
- 点击事件可在画布上叠加显示事件数据（房间图片等）
- 帮助验证 GUI 与事件脚本的配合

## 11. Button Effects 编辑器

菜单：工具 → Button Effects 编辑器

- 左侧：浏览模组和游戏的 `common/button_effects/*.txt` 文件
- 中间：选择要编辑的效果名称
- 右侧：三个标签页分别编辑 `potential`、`allow`、`effect` 代码块
- 支持新建、重命名、删除效果
- **Ctrl+S** 保存

---

## 12. GFX 精灵注册生成器

菜单：工具 → 生成 .gfx 精灵注册文件

格式：每行 `精灵名|贴图路径|类型|帧数`

```
GFX_my_button|gfx/interface/buttons/my_button.dds|spriteType|3
GFX_my_panel|gfx/interface/tiles/my_panel.dds|corneredTileSpriteType|1
```

右侧实时预览生成的 `.gfx` 代码，点击**保存**写入文件。

## 14. 虚拟编组

## 17. 常见问题

**Q：精灵图显示为棋盘格（无法加载）**
- 检查游戏目录是否正确（应包含 `interface/` 子目录）
- 部分 BC7 格式 DDS 需要 ffmpeg：在系统中安装 ffmpeg 并确保在 PATH 中
- 或安装 Microsoft texconv.exe 并设置 `TEXCONV_PATH` 环境变量

**Q：保存的 .gui 文件游戏不认**
- 确认文件位于 Mod 的 `interface/` 目录
- 确认文件名不含中文或特殊字符
- 验证工具：工具 → 验证当前 GUI 文件

**Q：中文名称/文字如何在游戏中显示？**
- 在 `.yml` 本地化文件中定义键值对
- 属性面板的文本字段填写本地化键（如 `MY_TITLE`），游戏会自动替换

**Q：程序启动后白屏/崩溃**
- 查看日志：`%USERPROFILE%\.stellaris_gui_editor\logs\stellaris_gui_editor.log`
- 向 GitHub 提交 Issue 并附上日志内容

**Q：打包后的 EXE 被杀毒软件误报**
- 已在构建配置中关闭 UPX 压缩（常见误报原因）
- 若仍被误报，请将整个 `StellarisGUIEditor/` 目录添加到杀毒白名单
